# A2A Go SDK - Developer Overview

## 1. Project Overview
The A2A Go SDK is a foundational library for building and consuming agentic applications using the **Agent2Agent (A2A) Protocol**. 
**Primary Problem Solved:** It abstracts protocol complexities (transports, serialization, state syncing, event streaming), allowing developers to focus solely on core business agent logic and execution.

## 2. Architecture & Patterns
The SDK employs a **layered, transport-agnostic architecture**:
*   **Domain Isolation**: Core types (Tasks, Messages) exist independently of wire transmission methods.
*   **Handler Pattern**: Server-side agent logic (`AgentExecutor`) is decoupled from the transport layer. Logic is wrapped in protocol-specific handlers (REST, JSON-RPC, or gRPC).
*   **Event-Driven Streaming**: Real-time observability through asynchronous events for artifact updates, task lifecycle changes, and agent thoughts mapping to A2A specs.

## 3. Directory Map
*   **`a2a/`**: **Core Domain.** Universal primitives (`Task`, `Message`, `Event`) and their constructors (e.g., `a2a.NewMessage(...)`). Single source of truth.
*   **`a2aclient/`**: **Client SDK.** High-level API structs and interfaces to connect to and interact with A2A servers.
*   **`a2asrv/`**: **Server SDK.** Interfaces for server execution (`AgentExecutor`), handler patterns, middleware, and non-gRPC transports (REST, JSON-RPC).
*   **`a2agrpc/` & `a2apb/`**: **gRPC Transport.** Logic for A2A over gRPC and underlying Protobuf definitions (`v1/pbconv/` for type conversions).
*   **`internal/`**: **Private Mechanisms.** Task lifecycle, state machine configuration (`taskexec/`), and task persistence adapters (`taskstore/`).
*   **`e2e/` & `examples/`**: End-to-end tests and reference implementations (e.g., `helloworld`).
*   **`docs/`**: Documentation, including AI developer guides (`ai/`).

## 4. Core Concepts
*   **AgentExecutor**: The core interface encapsulating the autonomous entity performing the work.
*   **Task (`TaskID`)**: A stateful unit of work or multi-turn conversation maintaining its execution state via `History`.
*   **Context (`ContextID`)**: A broader identifier securely linking related tasks (e.g., a user session or long-running project).
*   **Message**: A single unit of communication representing turns between `User` and `Agent`. It is composed of `Parts` (text, raw bytes, structured data).
*   **Artifact**: A distinct resource (e.g., a report, file) incrementally generated by the agent during task execution.
*   **Event**: Pervasive streaming primitives (like `TaskStatusUpdateEvent` or `TaskArtifactUpdateEvent`) used to distribute real-time state synchronization over the wire.
